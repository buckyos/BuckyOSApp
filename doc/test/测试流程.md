# BuckyOSApp 测试流程

本文档用于把测试目标拆成可执行的测试项。每个测试项都包含：

- 大纲标题
- 场景描述
- 测试通过标准

执行测试时，优先以 UI 检查点和截图作为通过依据。

## 大纲

1. 创建账户测试
2. 导入账户测试
3. OOD 查找与激活测试
4. Active DID 切换一致性测试
5. 异常路径测试
6. BuckyApi 关键接口测试
7. 安全审查
8. 日志扫描
9. 持久化数据审查
10. 测试报告输出

## 1. 创建账户测试

### 场景描述

验证用户可以完成“创建 DID + 绑定 SN”的主流程，并且在用户名不可用、激活码不可用时可以在当前流程内修改并继续重试。

### 测试通过标准

- 能从欢迎页进入创建流程。
- 能完成助记词展示与确认。
- 当用户名不可用时，页面出现明确提示，且仍允许修改后继续提交。
- 当激活码不可用时，页面出现明确提示，且仍允许修改后继续提交。
- 最终能到达创建成功页。
- 存在创建成功截图，可作为通过证据。

建议截图：

- `01-create-success.png`

## 2. 导入账户测试

### 场景描述

验证用户可以使用固定助记词重新导入已创建过的身份，并且导入后的名称与创建时一致。

### 测试通过标准

- 能从欢迎页进入导入 DID 页面。
- 输入助记词和密码后可完成导入。
- 导入后的当前身份名称与创建成功时的名称一致。
- 不出现“无 SN 记录”或“重复导入”导致的误失败。
- 存在导入成功截图，可作为通过证据。

建议截图：

- `02-import-success.png`

## 3. OOD 查找与激活测试

### 场景描述

验证在已完成 SN 绑定的前提下，应用可以扫描局域网设备，发现目标 OOD，并成功打开或完成激活流程。

### 测试通过标准

- 能进入 OOD 激活入口页。
- 扫描流程可以启动并结束。
- 设备列表中能出现目标 OOD，或明确记录扫描失败的卡点。
- 点击设备后能打开设备返回的激活页。
- 若激活页支持完成绑定，则可以到达激活成功检查点。
- 存在 OOD 激活成功截图，或存在阻塞截图并在报告中说明卡点。

建议截图：

- `03-ood-activated.png`
- 若失败：`03-ood-blocked.png`

## 4. Active DID 切换一致性测试

### 场景描述

验证切换当前身份后，首页、SN 状态、OOD 状态以及宿主暴露给页面的身份信息都同步切换，不出现旧身份残留。

### 测试通过标准

- 能在身份列表中切换 Active DID。
- 切换后首页显示的新名称与目标 DID 一致。
- SN 状态和 OOD 状态随新 DID 切换。
- `BuckyApi.getCurrentUser()` 返回的 DID、用户名、`sn_username` 与当前 Active DID 一致。
- 存在切换后界面截图或接口返回截图，可作为通过证据。

建议截图：

- `04-active-did-switched.png`

## 5. 异常路径测试

### 场景描述

验证主链路外的重要失败分支是否有正确反馈，且不会造成状态错乱或敏感信息泄露。

### 测试通过标准

- 删除身份输入错误密码时，会提示错误且允许重试。
- 备份助记词输入错误密码时，不会展示助记词。
- 导入本地已存在 DID 时，会被正确拦截。
- 导入无 SN 记录的 DID 时，会明确失败。
- 取消交互式操作时，会返回取消结果，不会卡死。
- 每个失败分支都至少有一个 UI 截图或报错截图作为证据。

建议截图：

- `05-invalid-password.png`
- `05-import-duplicate-blocked.png`
- `05-import-no-sn-record-blocked.png`

## 6. BuckyApi 关键接口测试

### 场景描述

验证宿主注入的 `BuckyApi` 在关键接口上的返回结构、成功结果和错误分支符合当前文档约定。

### 测试通过标准

- `getPublicKey` 可以成功返回公钥，或在无公钥时返回正确错误码。
- `getCurrentUser` 返回的 DID、用户名、`public_key`、`sn_username` 正确。
- `signJsonWithActiveDid` 成功时返回 `signatures`。
- `signJsonWithActiveDid` 成功时返回 `pwd_hash`，且行为符合当前约定。
- 密码错误、用户取消、并发 busy 的结果与文档一致。
- 至少存在一张调试页或返回结果截图作为通过证据。

建议截图：

- `06-buckyapi-sign-success.png`
- `06-buckyapi-current-user.png`

## 7. 安全审查

### 场景描述

审查当前实现是否存在私钥、助记词、密码相关的高风险问题，尤其关注前端桥接、后端存储、接口返回和删除流程。

### 测试通过标准

- 未发现私钥明文落盘。
- 未发现助记词明文持久化。
- 未发现密码明文持久化。
- 未发现可直接导致以上信息泄露的高风险接口或实现。
- 审查结论写入测试报告，并列出发现项或“未发现高风险”。

证据形式：

- 代码审查记录
- 命令搜索结果
- 报告中的审查结论

## 8. 日志扫描

### 场景描述

检查前端与后端日志输出中是否包含助记词、密码、私钥、完整 `pwd_hash` 等敏感信息。

### 测试通过标准

- 未发现直接打印助记词、密码、私钥的日志。
- 未发现会在错误或调试输出中泄露敏感字段值的日志。
- 若发现可疑日志，已在报告中列出文件和位置。

证据形式：

- 日志搜索命令结果
- 报告中的日志扫描结论

## 9. 持久化数据审查

### 场景描述

检查本地存储、DID 存储和 SN 状态缓存中，是否保存了不该保存的敏感信息，或是否存在删除后状态串用问题。

### 测试通过标准

- 未发现助记词明文持久化。
- 未发现私钥明文持久化。
- 未发现密码明文持久化。
- 删除 DID 后，本地缓存和持久化状态不会串到其他 DID。
- 审查结论写入测试报告，并说明是否存在残留风险。

证据形式：

- 代码审查记录
- 存储实现检查结果
- 报告中的持久化审查结论

## 10. 测试报告输出

### 场景描述

将本轮测试结果汇总为统一格式的测试报告，便于复查、回归和跨 agent 交接。

### 测试通过标准

- 已生成测试报告。
- 报告中包含每个测试项的结果。
- 报告中包含通过截图或阻塞截图路径。
- 若未通过，报告中明确写出卡在哪一步、看到什么现象、是否可重试。

建议输出文件：

- `doc/test/测试报告模板.md`
- 实际测试报告保存到执行约定目录
