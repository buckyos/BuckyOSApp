# Android 端到端测试流程：创建、导入并重新绑定 OOD

本文档描述在全新 Android 测试设备中，从首次启动 App、创建账户、绑定 SN、绑定 OOD，到删除账户后重新导入，再解除绑定 OOD、重置 OOD 并重新绑定的端到端验收流程。

本流程使用真实 SN 环境。Active Code 由测试负责人按次提供未使用值，不在仓库文档、截图、日志或报告中记录完整明文。

## 0. 测试参与者需提供

执行前，测试参与者需要提供或确认以下内容：

- 待测 APK：询问使用哪个 APK；如果测试参与者不指定，默认使用当前项目代码构建 Android 包。
- 测试设备：询问使用什么设备；默认使用当前已连接的 Android 实体机，没有实体机时使用 Pixel 4 XL 模拟器。
- OOD 重置权限：询问是否允许自动重置 OOD；默认允许。报告只记录“OOD 已恢复为可绑定状态”，不记录内部操作细节。
- SN 环境：询问 SN 环境是否可用；默认视为可用。
- 可用邀请码：测试参与者必须提供一个未使用的 Active Code。测试期间还需要使用无效邀请码验证错误提示，并保留显示无效的关键截图。
- 可用用户名：由测试流程自动生成，默认以 `ad-autotest` 开头。

本流程的自动化等级：可部分自动化。

原因：Android App 安装、清数据、点击、输入和关键截图采集可以自动化；但真实 SN 的 Active Code 可用性、真实 OOD 网络发现、OOD 激活页内部流程和最终绑定状态刷新依赖外部环境，不能保证完全由脚本稳定控制。

## 1. 测试目标

验证全新 Android 版本满足以下端到端结果：

1. 清空数据后首次启动进入欢迎页。
2. 用户可以完成创建账户流程，包括助记词生成、助记词确认、SN 用户名与 Active Code 校验、SN 注册、owner key 绑定和本地 DID 创建。
3. 创建完成后的 DID 自动成为 Active DID。
4. 用户可以进入 OOD 激活入口，扫描局域网内可绑定 OOD。
5. 用户可以打开 OOD 激活页并完成绑定。
6. 绑定完成后 App 能展示 OOD 已绑定状态，并在重启后保持一致。
7. 用户可以删除当前本地账户，再用第一阶段备份的助记词重新导入同一 SN 账户。
8. 导入后仍能识别 SN 用户名和 OOD 绑定状态。
9. 用户可以解除 OOD 绑定，将 OOD 重置为可绑定状态后，再次完成 OOD 绑定。

## 2. 前置条件

- Android 设备：优先使用已连接实体机；没有实体机时使用 Pixel 4 XL 模拟器。设备需可通过 adb 访问。
- App：安装本轮待测 Android 包。若测试机已有旧版本，先尝试覆盖安装；覆盖安装不成功时，卸载旧版本后安装新版本。测试前清理旧 App 数据。
- SN：使用真实 SN 服务，`sn_host` 指向真实 SN base URL。
- 测试账号数据：
  - 用户名：自动生成，格式为 `ad-autotest-${随机6个字母}-${时间戳}`，例如 `ad-autotest-abcdef-20260519123045`。
  - 密码：`111111`。
  - Active Code：由测试负责人按次提供未使用值。
- OOD：已接入与 Android 测试设备可互通的局域网。执行 4.1 前必须使用本机 `reset-ood-bindable` skill 重置一次 OOD，使其处于可绑定状态；报告只记录准备结果，不记录内部操作细节。
- Android 执行工具：推荐使用 [$test-android-apps:android-emulator-qa]执行设备安装、输入和关键截图采集。

## 3. 报告输出

每轮测试输出到独立目录：

```
auto_test/runs/YYYYMMDD-HHMMSS/android-e2e-{pass|blocked}-{简短结果}/
```

报告内容只保留：

- 结果报告 Markdown：测试摘要、每个阶段的通过/失败结果、流程中的异常、最终结论。
- `screenshots/`：关键页面截图。

截图命名：

- 所有截图文件名前必须带时间戳，格式为 `YYYYMMDD-HHMMSS-<步骤编号>-<说明>.png`。
- 示例：`screenshots/20260520-143012-01-welcome.png`。
- 文档步骤里的截图文件名是语义示例，实际报告必须使用带时间戳的最终文件名。

报告正文必须写明：

- 本轮自动生成的 `username`。
- 本轮使用的 `active_code` 脱敏值，格式建议保留首尾，例如 `abcd...wxyz`。
- 不需要为 username 或 active_code 单独创建记录文件。

阶段成果必须把步骤、结论、说明和关键截图对应起来，例如：

```text
4.1 准备 Android 全新状态：PASS。安装本轮 APK、清空 App 数据后进入欢迎页。（screenshots/20260520-143012-01-welcome.png）
```

报告文件名必须直接体现测试结果：

- 全流程通过：`PASS-android-e2e-create-import-rebind-ood.md`
- 中途卡住：`BLOCKED-step-<步骤编号>-<简短原因>.md`
- 明确失败：`FAIL-step-<步骤编号>-<简短原因>.md`

不需要在报告目录中保存 adb 日志、logcat、SN 请求日志或 UI tree。报告和截图不得包含完整助记词、明文密码、私钥 PEM、完整 `pwd_hash`、完整 Active Code 或完整 SN token。

## 4. 端到端步骤

### 4.1 准备 Android 全新状态

操作：

1. 使用本机 `reset-ood-bindable` skill 重置一次 OOD，使 OOD 进入可绑定状态。
2. 启动或连接 Android 测试设备。
3. 检查测试机是否已安装旧版本 App。
4. 如已安装旧版本，先尝试覆盖安装本轮待测 Android 包。
5. 如果覆盖安装不成功，卸载旧版本后安装本轮待测 Android 包。
6. 清理 App 数据，确保没有本地 DID、Active DID、SN 状态缓存和历史 localStorage。
7. 启动 App。

验证：

- 当前页面是欢迎页。
- 页面存在“创建账户”和“导入账户”入口。
- 不直接进入主应用、身份切换页或历史流程页。
- 如测试机原本有旧版本，最终运行的必须是本轮待测新版本。

通过标准：

- 欢迎页稳定显示，入口可点击。
- OOD 准备结果记录为“已恢复为可绑定状态”。
- 截图保存为 `screenshots/YYYYMMDD-HHMMSS-01-welcome.png`。

### 4.2 进入创建账户流程

操作：

1. 在欢迎页点击“创建账户”。
2. 进入创建账户说明页。
3. 检查 DID 说明卡片和 SN 说明卡片。
4. 点击主按钮进入助记词展示页。

验证：

- 创建账户说明页没有要求输入密码、昵称或 Active Code。
- 页面说明表达“先准备 DID 材料，再完成 SN 绑定”的流程。
- 点击主按钮后生成助记词并进入助记词展示页。

通过标准：

- 助记词展示页出现 12 个英文单词。
- 截图保存为 `screenshots/YYYYMMDD-HHMMSS-02-create-intro.png` 和 `screenshots/YYYYMMDD-HHMMSS-03-mnemonic-show.png`。

### 4.3 备份并确认助记词

操作：

1. 临时读取并备份 12 个助记词，用于后续删除账户后的导入验证。
2. 点击确认进入助记词确认页。
3. 按展示顺序依次点击词池中的助记词。
4. 点击继续进入绑定 SN 页。

验证：

- 助记词为 12 个英文 BIP-39 单词。
- 助记词确认页必须按正确顺序完成。
- 顺序未完成前不能进入绑定 SN 页。

通过标准：

- 正确确认后进入绑定 SN 页。
- 报告不记录完整助记词。
- 助记词必须在测试执行期间可用于 4.13 导入账户；测试结束后清理临时记录。
- 截图保存为 `screenshots/YYYYMMDD-HHMMSS-04-mnemonic-confirm.png`。

### 4.4 验证 SN 无效输入

操作：

1. 在绑定 SN 页输入无效用户名，例如 `ad_autotest_bad`。
2. 输入密码 `111111` 和确认密码 `111111`。
3. 暂不输入可用 Active Code，等待页面展示用户名格式错误提示，并保存截图。
4. 自动生成本轮可用用户名，格式为 `ad-autotest-${随机6个字母}-${时间戳}`，并输入该用户名。
5. 输入无效邀请码，例如 `BAD_ACTIVE_CODE`。
6. 等待页面展示 Active Code 不可用或校验失败提示，并保存截图。

验证：

- 无效用户名不能通过本地校验。
- 无效邀请码不能通过远端校验。
- 提交按钮不可用，或点击后不能提交创建。
- 页面分别明确显示用户名无效和 Active Code 无效提示。

通过标准：

- 保留无效用户名提示截图 `screenshots/YYYYMMDD-HHMMSS-05-invalid-username.png`。
- 保留无效邀请码提示截图 `screenshots/YYYYMMDD-HHMMSS-06-invalid-active-code.png`。

### 4.5 填写并校验可用 SN 信息

操作：

1. 沿用 4.4 中生成的可用用户名。
2. 确认用户名仍为该可用用户名。
3. 保持密码 `111111` 和确认密码 `111111`。
4. 清空无效邀请码并输入测试参与者提供的可用 Active Code。
5. 等待用户名和 Active Code 远端校验完成。

验证：

- 用户名输入会规范化为小写。
- 用户名以 `ad-autotest` 开头，且符合 `^[a-z][a-z0-9-]{5,}[a-z0-9]$`。
- 密码长度满足规则，两次密码一致。
- Active Code 非空。
- 用户名远端检查成功。
- Active Code 远端检查成功。
- 校验完成前提交按钮不可用或不能提交。

通过标准：

- 提交按钮进入可点击状态。
- 页面没有显示用户名不可用、Active Code 无效或密码不一致错误。
- 截图保存为 `screenshots/YYYYMMDD-HHMMSS-07-bind-sn-ready.png`。

### 4.6 提交创建账户并完成 SN 绑定

操作：

1. 点击提交。
2. 等待创建流程完成。
3. 观察页面是否进入创建成功页。

验证：

- 页面从绑定 SN 流程进入创建中状态。
- 最终进入创建成功页，而不是停留在绑定页或错误提示页。
- 成功页展示本轮自动生成的用户名或对应昵称。
- 成功页展示 BuckyOS DID。
- 本地 DID 不应在 SN 绑定失败时提前完成创建。

通过标准：

- App 进入创建成功页。
- 创建成功页展示昵称和 BuckyOS DID。
- 当前 DID 自动成为 Active DID。
- 截图保存为 `screenshots/YYYYMMDD-HHMMSS-08-create-success.png`。

### 4.7 进入 OOD 激活入口

操作：

1. 在创建成功页点击主按钮进入 OOD 激活入口。
2. 如未自动进入，手动进入 `/main/home/ood-activate` 对应页面。

验证：

- 页面展示 OOD 激活标题、说明和插图。
- 展示“扫描本地网络内的设备”入口。
- 远程激活入口如存在，应为禁用或不可用状态。
- TabBar 显示符合主应用页面规则。

通过标准：

- 当前账号处于未绑定 OOD 状态。
- 扫描入口可点击。
- 截图保存为 `screenshots/YYYYMMDD-HHMMSS-09-ood-activate.png`。

### 4.8 扫描可绑定 OOD

操作：

1. 确认 OOD 已处于可绑定状态并与 Android 测试设备网络互通。
2. 点击“扫描本地网络内的设备”。
3. 等待扫描完成。
4. 在扫描结果中选择目标 OOD。

验证：

- 扫描页显示扫描中状态和进度。
- 扫描完成后只展示返回有效 `active_url` 的设备。
- 目标 OOD 的设备名或 IP 可识别。
- 无效设备、超时设备不应作为可绑定目标展示。

通过标准：

- 目标 OOD 出现在待激活设备列表。
- 点击目标设备后进入 `/web-container?embedded=1` 承载的 OOD 激活页。
- 截图保存为 `screenshots/YYYYMMDD-HHMMSS-10-ood-scan-result.png`。

### 4.9 在 OOD 激活页完成绑定

操作：

1. 在 OOD 激活页按页面提示继续绑定。
2. 如页面要求确认当前身份，确认显示的是本轮创建的 SN 用户名或 DID。
3. 如页面要求签名或密码确认，输入本轮统一密码 `111111`。
4. 等待 OOD 页面返回绑定成功结果。
5. 返回 App 首页或 OOD 状态页。

验证：

- OOD 激活页在 App WebView 容器内打开，不跳出 Android App。
- 签名请求使用当前 Active DID。
- 密码弹窗不泄露明文密码。
- 绑定成功后，SN 或 OOD 状态可被刷新到当前 DID。

通过标准：

- OOD 页面显示绑定成功，或 App 返回后显示 OOD 已绑定。
- 截图保存为 `screenshots/YYYYMMDD-HHMMSS-11-ood-bind-success-page.png`。
- 如果出现签名弹窗，保存弹窗截图但不得记录密码。

### 4.10 验证 App 展示 OOD 已绑定

操作：

1. 回到 `/main/home` 或 `/main/home/ood-activate`。
2. 刷新或重新进入页面，等待绑定状态加载完成。
3. 检查首页 OOD 区域。

验证：

- 当前页面展示 OOD 已绑定状态。
- 展示进入应用页按钮。
- 展示解除绑定入口。
- 不再展示“扫描本地网络内的设备”作为主引导状态。
- 当前 DID 的 SN 状态包含非空 `zone_config` 或等效 OOD 绑定状态。

通过标准：

- UI 明确显示 OOD 已绑定。
- 截图保存为 `screenshots/YYYYMMDD-HHMMSS-12-ood-bound.png`。

### 4.11 重启 App 后复核持久化状态

操作：

1. 关闭 App。
2. 重新启动 App。
3. 检查启动后的默认页面。
4. 进入首页和 OOD 状态页。

验证：

- App 不回到欢迎页。
- App 自动使用本轮创建的 Active DID。
- 首页显示当前 SN 用户名或身份信息。
- OOD 状态仍为已绑定。

通过标准：

- 重启后仍进入主应用。
- OOD 已绑定状态保持。
- `BuckyApi.getCurrentUser()` 如可验证，应返回当前 DID、公钥、用户名和 SN 用户名，且与页面展示一致。
- 截图保存为 `screenshots/YYYYMMDD-HHMMSS-13-after-restart-bound.png`。

### 4.12 删除当前本地账户

操作：

1. 从主应用进入设置页。
2. 进入删除账户流程。
3. 在风险确认弹窗中先取消一次，确认取消不会删除账户。
4. 再次进入删除账户流程并确认。
5. 输入当前账户密码 `111111`。
6. 等待删除完成。

验证：

- 取消删除后仍停留在当前账户，首页和 OOD 已绑定状态不变化。
- 确认删除且密码正确后，本地账户被删除。
- 删除最后一个本地 DID 后，App 回到欢迎页。
- 删除本地账户不应清除真实 SN 记录，也不应让已备份助记词失效。

通过标准：

- 删除取消分支正常。
- 删除确认后回到欢迎页。
- “创建账户”和“导入账户”入口可见。
- 截图保存为 `screenshots/YYYYMMDD-HHMMSS-14-delete-cancel.png` 和 `screenshots/YYYYMMDD-HHMMSS-15-delete-back-welcome.png`。

### 4.13 使用第一阶段助记词重新导入账户

操作：

1. 在欢迎页点击“导入账户”。
2. 输入 4.3 中备份的 12 个助记词。
3. 输入密码 `111111`。
4. 输入确认密码 `111111`。
5. 提交导入。
6. 等待导入完成并进入主应用。

验证：

- 导入页不要求输入 SN 用户名或 Active Code。
- App 能根据助记词派生公钥，并从真实 SN 查询到该账户记录。
- 导入成功后，本地昵称或身份信息使用第一阶段创建的 SN 用户名。
- 导入成功后的 DID 自动成为 Active DID。
- 导入后 OOD 状态应能识别为已绑定，或在刷新后展示已绑定状态。

通过标准：

- 成功进入主应用。
- 首页显示第一阶段创建的用户名或对应身份信息。
- OOD 状态为已绑定。
- 截图保存为 `screenshots/YYYYMMDD-HHMMSS-16-import-form.png`、`screenshots/YYYYMMDD-HHMMSS-17-import-success-home.png` 和 `screenshots/YYYYMMDD-HHMMSS-18-import-ood-bound.png`。

### 4.14 解除绑定 OOD

操作：

1. 在导入后的账户首页进入 OOD 已绑定状态页。
2. 点击解除绑定入口。
3. 在确认弹窗中先取消一次，确认状态不变化。
4. 再次点击解除绑定并确认。
5. 如需要签名或密码确认，输入密码 `111111`。
6. 等待解除绑定完成。

验证：

- 取消解绑后仍显示 OOD 已绑定。
- 确认解绑后，App 回到 OOD 未绑定或激活入口状态。
- 解绑失败时，不应错误显示为未绑定。
- 解除绑定只解除当前 DID 与 OOD 的绑定关系，不应删除本地账户。

通过标准：

- 解绑取消分支正常。
- 确认解绑后，当前账号仍存在，但 OOD 状态变为未绑定。
- 截图保存为 `screenshots/YYYYMMDD-HHMMSS-19-unbind-cancel.png` 和 `screenshots/YYYYMMDD-HHMMSS-20-unbind-success.png`。

### 4.15 重置 OOD 并重新绑定

操作：

1. 使用本机 `reset-ood-bindable` skill 再次重置 OOD，使其恢复为可绑定状态。
2. 回到 App 的 OOD 激活入口。
3. 点击“扫描本地网络内的设备”。
4. 在扫描结果中选择目标 OOD。
5. 在 OOD 激活页按页面提示完成绑定。
6. 如页面要求签名或密码确认，输入密码 `111111`。
7. 返回 App 首页或 OOD 状态页。

验证：

- 报告只记录“OOD 已恢复为可绑定状态”，不记录重置内部细节。
- 重置后扫描仍能发现目标 OOD。
- OOD 激活页仍在 Android App WebView 容器中打开。
- 当前绑定身份是导入后的 Active DID。

## 6. 失败记录要求

如果流程失败，报告必须写明：

- 失败步骤编号。
- 实际页面现象。
- 卡在哪一个操作，用户或脚本当时正在等待什么结果。
- 可能原因，例如真实 SN、Active Code、真实 OOD、局域网、OOD 重置状态、WebView 页面或 App 状态刷新。
- 是否依赖真实 SN、真实 OOD、局域网或 Active Code 状态。

即使中途卡住，也必须输出 `BLOCKED-step-<步骤编号>-<简短原因>.md` 报告，并保留卡住页面截图。

常见阻塞项：

- Active Code 已被使用或真实 SN 返回不可用。
- 用户名已被占用。
- Android 测试设备与 OOD 不在可互通网络内。
- OOD 未处于可绑定状态。
- OOD 激活页打开但绑定流程未返回成功。
- 绑定成功后 SN 状态刷新延迟，App 暂未显示 `zone_config`。
- 删除账户后未回到欢迎页。
- 助记词导入失败或导入后未恢复 SN 用户名。
- OOD 解绑后状态未刷新，导致无法重新绑定。
